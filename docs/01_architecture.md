Разделение ответственности простое: `browser.py` умеет только доставать слоты и
ничего не знает про уведомления; `alerts/` умеет только доставлять и ничего не
знает про Госуслуги; `main.py` решает, что считать событием.

## Один цикл опроса

```mermaid
sequenceDiagram
    participant M as SlotMonitor
    participant W as Watcher
    participant P as Страница Chrome
    participant E as ЕПГУ
    participant D as Каналы

    loop по каждому отделению
        M->>W: fetch_slots(отделение)
        W->>P: page.evaluate(fetch)
        P->>E: POST /equeue/agg/slots<br/>куки, X-Order-Id, X-Form-Id
        E-->>P: {"slots": [...]}
        P-->>W: {status, payload}
        alt 401 или 403
            W-->>M: SessionLost
        else 200
            W-->>M: SlotsResponse
            alt слоты есть и повтор не подавлен
                M->>D: Alert(kind="slots")
            else пусто
                M->>M: запомнить «слотов нет»
            end
        end
        M->>M: пауза 1–4 с
    end
    M->>W: save_state — сохранить куки
    M->>M: пауза period ± jitter
```

Запрос выпускается не из Python, а из самой страницы через `page.evaluate`.
Это принципиально: так он несёт подлинные куки, User-Agent и TLS-отпечаток
Chrome — расходиться нечему. Подробнее в [03_session.md](03_session.md).

## Когда поднимать шум

Наличие слотов ещё не событие: если они висят час, а сторож обходит отделения
каждые 8 минут, без подавления вы получите семь одинаковых сигналов.

```mermaid
flowchart TD
    START["Ответ по отделению"] --> STATUS{"HTTP 200?"}
    STATUS -->|"401 / 403"| LOST["SessionLost<br/>цикл прерывается"]
    STATUS -->|"другой код"| ERR["Alert: error"]
    STATUS -->|"200"| HAS{"Слоты есть?"}

    HAS -->|"нет"| CLEAR["Запомнить: слотов нет<br/>тишина"]
    HAS -->|"да"| PREV{"В прошлый раз<br/>слоты были?"}

    PREV -->|"нет — появились"| FIRE["Alert: slots"]
    PREV -->|"да"| AGE{"Прошло больше<br/>repeat_alert_min?"}
    AGE -->|"да"| FIRE
    AGE -->|"нет"| QUIET["Тишина:<br/>сигнал был недавно"]

    FIRE --> MARK["Запомнить время сигнала"]
```

Состояние живёт в двух словарях `SlotMonitor`, по коду отделения: было ли в
прошлый раз пусто и когда в последний раз шумели. Оно у каждого отделения своё —
слоты в Марфино не гасят сигнал по Марьиной Роще.

Важная деталь: **пустой ответ — не ошибка**. Портал отвечает `200` и
`"Operation completed"` с пустым списком, это штатное «мест нет».

## Живучесть

Ошибка одного отделения не должна ронять обход, а сбой доставки — мониторинг.
Поэтому исключения гасятся на трёх уровнях, и на каждом — по своей причине:

```mermaid
flowchart LR
    A["Отделение<br/>упало"] --> A1["Предупреждение в лог,<br/>идём к следующему"]
    B["Канал<br/>упал"] --> B1["Предупреждение в лог,<br/>остальные каналы работают"]
    C["Сессия<br/>протухла"] --> C1["Alert, ждём входа,<br/>цикл продолжается"]
```

Единственное исключение, которое намеренно пробрасывается наверх, —
`SessionLost`: продолжать обход бессмысленно, пока не вошли заново.

## Что происходит при запуске

```mermaid
stateDiagram-v2
    [*] --> Настройки
    Настройки --> Каналы: .env прочитан и проверен
    Каналы --> Браузер: список каналов собран
    Браузер --> Вход: профиль открыт, куки восстановлены
    Вход --> Опрос: сессия активна
    Опрос --> Опрос: слоты проверены, пауза
    Опрос --> Вход: SessionLost
    Настройки --> [*]: ошибка в .env, код возврата 2
```

Проверка настроек идёт **до** запуска Chrome: незачем открывать окно и просить
логин, чтобы потом сообщить об опечатке в номере заявки.

## Режим репетиции

```bash
uv run gswatch --selftest
```

Прогоняет семь сцен на подставных ответах портала, но через настоящие каналы.
Подмена — только на границе с ЕПГУ:

```mermaid
flowchart LR
    SELF["selftest.py<br/>заготовленные ответы"] --> MON["SlotMonitor<br/>тот же, что в бою"]
    MON --> DISP["dispatch"]
    DISP --> REAL["настоящие каналы:<br/>SMS, Telegram, Мак"]
```

Гоняется именно боевой `SlotMonitor`, а не копия логики — иначе проверка
проверяла бы саму себя. Заодно видно то, что вживую ловилось бы месяцами:
что подавление повторов работает и что шум поднимается только на слоты.
